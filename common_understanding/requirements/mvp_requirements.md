# MVP Requirements

日付: 2026-05-27

## MVPの方向性

MVPは「応用情報の学習を進めるためのTodo・学習ログ・頑張り可視化ダッシュボード」とする。

英語日記機能は既存資産として残してよいが、MVPの主役はTodoと学習支援に寄せる。

## トップ画面

トップ画面では、大タスクが見やすく整理されている状態を重視する。

大タスクの詳細を開くとサブタスクが見える。さらに、数値が決まっている明確な小タスクはトップ画面にも表示されているとよい。

### 初期ウィジェット候補

1. Todo
2. 可視化された学習度合い
3. ショートカット
4. ログ・日記入力
5. AI秘書パネル

AI秘書パネルは、今日の状況を短く要約し、次にやるとよさそうなことを提示する領域とする。

## Todo要件

### タスク階層

- 大タスク: 応用情報
- 実行単位: 過去問道場、模擬試験、苦手復習など
- サブ分類: ネットワーク、マネジメント、セキュリティなどの出題ジャンル
- 明確な数値を持つ小タスク: 問題数、学習分数、模擬試験回数など

### Todo項目

- タイトル
- 優先度
- カテゴリ
- 実績時間
- メモ
- リンク
- 完了ステータス

### ステータス候補

- 準備中
- 未着手
- 進行中
- 完了
- 保留
- スキップ

### 優先度候補

MVPでは `今日中`、`なるべく早く`、`余裕があれば` のような行動に近い表現を優先候補にする。

内部値としては `high`、`medium`、`low` にマッピングしてもよい。

### 将来検討

- 期限
- 見積時間
- 繰り返し
- タスク依存関係
- プロジェクト連携

## 頑張り可視化

### 指標候補

- 完了タスク数
- 学習時間
- 継続日数
- AI評価

学習時間と継続日数を主指標にする。完了タスク数は補助指標にする。

### 注意

AI評価はモチベーションを下げない形にする。厳しい採点より、秘書・分析官としての客観的な整理と肯定的な改善提案を重視する。

## 学習ログ

### 単位

学習ログはセッションごとに残す。

### 初期ジャンル

- 応用情報

### 初期サブカテゴリ候補

- 過去問道場
- 模擬試験
- 苦手復習
- 知識整理

### 初期出題ジャンル候補

- セキュリティ
- ネットワーク
- データベース
- マネジメント
- ストラテジ
- システム開発
- アルゴリズム
- その他

### 検索・分類

- キーワード検索
- ジャンル検索
- タグ検索
- 日付検索は余裕があれば対応

### 入力項目

- 実績時間
- 分類
- 出題ジャンル
- タグ
- メモ
- 関連Todo

### 自動補完

理想は自動判別・自動入力。MVPではルールベースで分類や出題ジャンルの候補を補完し、将来的にAIによる分類・タグ付けへ拡張する。

## UIカスタマイズ

### MVPで扱う範囲

- 色テーマ
- ウィジェット表示・非表示
- ウィジェット並び替え
- タグ追加
- ジャンル追加
- Todoカテゴリ追加

### 最終形の方向性

Notionのように好きな場所へ表やウィジェットを置ける柔軟なダッシュボード編集を目指す。

ただしMVPでは、完全な自由配置ではなく、固定レイアウト内でのカスタマイズに留める。

## ショートカット

ショートカットは、このアプリを起動台にするための重要機能とする。

### 候補

- 教材サイト
- テックニュース
- TED
- IT時事ニュース
- カフェ・自然スポット一覧
- 開発プロジェクト
- GitHub
- VS Code起動
- プロジェクト関連Webサイト
- プロジェクトごとの環境
- 連携するNotionページ
- ナレッジ蓄積ページ

### 注意

GitHub Pages上の静的フロントからローカルのVS Codeを直接起動するには制約がある。MVPではまずWebリンク中心にし、ローカルアプリ起動は将来検討とする。

## 個人情報

MVPでは以下を保存しない、または保存対象から明確に外す。

- 住所
- 実名
- 仕事上の機密
- 一般的な個人情報の詳細

収入・支出などのセンシティブな領域は将来候補だが、MVPでは優先しない。

## UI刷新

既存画面は大きく作り替えてよい。英語日記は必要であれば1ウィジェットまたはサブ機能として残す。

MVPでは、世界中で評価されている優れたダッシュボード・タスク管理・起動台UIを参考にしつつ、Todoと学習状況が主役になる画面へ刷新する。

## 保存先

MVPの第一候補はNotion。Todo、学習ログ、ショートカット、タグ、ジャンル設定をNotion DBに保存する方向で検討する。

ただし、設定がより簡単で、ダッシュボード上から管理しやすく、セキュリティ面でも問題が少ない代替手段があれば検討する。
