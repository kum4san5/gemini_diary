# Scoring Draft

## 基本方針

頑張りの可視化は、タスク数よりも学習時間と継続力を重視する。

完了タスク数は補助指標にする。タスクを細かく切りすぎることでスコアが不自然に上がる設計は避ける。

## 初期スコア案

```text
daily_score =
  学習時間ポイント
  + 継続ボーナス
  + 完了ボーナス
  + AI評価補正
```

## 学習時間ポイント

```text
min(学習分数, 180)
```

1日の上限は180点。長時間学習を評価しつつ、無理な長時間だけが正義にならないようにする。

## 継続ボーナス

```text
継続日数に応じて 0〜50 点
```

例:

- 1日: 5点
- 3日: 15点
- 7日: 30点
- 14日以上: 50点

## 完了ボーナス

```text
完了タスク数 × 5点
```

補助的な扱いにする。

## AI評価補正

AI評価は厳しい採点ではなく、学習ログの内容や継続状況を見た軽い補正に留める。

例:

- 記録が具体的: +5
- 苦手復習に取り組んだ: +5
- 無理な計画が続いている: 改善提案のみ。減点は原則しない。

## 表示方針

スコアだけでなく、以下も並べて表示する。

- 今日の学習時間
- 今週の学習時間
- 継続日数
- 最近の学習ジャンル
- AI秘書の短評
