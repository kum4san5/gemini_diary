# MVP Direction

## 現時点のMVP仮説

最初のMVPは、英語日記アプリの延長ではなく「応用情報学習とTodoを中心にした人生ダッシュボードの最小版」とする。

## MVPで目指す体験

朝に開くと、今日やることと自分の頑張り具合が見える。学習を始めやすく、完了したタスクや学習ログが蓄積される。後からジャンル別に探せるため、努力が消えずに資産化される。

大タスクが整理され、詳細を見るとサブタスクが確認できる。数値が明確な小タスクはトップ画面にも表示され、今すぐ取りかかれる状態にする。

## MVP候補機能

1. Todo管理
   - 今日のTodo
   - 完了チェック
   - 優先度
   - 学習タスクとの連携
   - 大タスク
   - サブタスク
   - 数値が明確な小タスク

2. 頑張り可視化
   - 今日の完了数
   - 連続記録
   - 学習時間または学習回数
   - 日別カレンダー表示
   - AI評価

3. 学習ログ
   - ジャンル選択
   - 短いメモ
   - 学習対象
   - 検索
   - タグ
   - セッション単位の記録

4. AI秘書・分析官
   - 今日のTodo整理
   - 学習ログの要約
   - 週次の頑張りまとめ
   - 次にやることの提案

## MVPでまだ入れない可能性があるもの

- 収入・支出管理
- 詳細な健康管理
- 複雑な目標管理
- 外部ツールの深い自動実行
- 高度な認証
- 完全なUIビルダー

## 次に決めること

- Todoのデータ構造
- 学習ログの分類軸
- 頑張り具合の指標
- トップ画面のウィジェット
- 保存先
- 個人情報を扱う範囲
- 応用情報向けの初期タスクテンプレート
- AI秘書パネルの役割
